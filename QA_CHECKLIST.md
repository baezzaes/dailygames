# Weekly Ranking QA Checklist

## 1) 허브(메인) UI
- [ ] 최초 진입 시 랭킹 토글 기본값이 `오늘`로 활성화된다.
- [ ] 허브 `오늘/주간` 토글 클릭 시 각 게임 카드 TOP3가 즉시 바뀐다.
- [ ] 토글을 여러 번 전환해도 PB 텍스트가 중복 누적되지 않는다.
- [ ] TOP3 닉네임 중 비속어/성적 표현은 `[filtered]`로 표시된다.
- [ ] 랭킹 데이터가 없을 때 카드 랭킹 영역이 깨지지 않고 비어있는 상태로 유지된다.

## 2) 게임 화면 UI (10개 게임 공통)
- [ ] 각 게임 랭킹 카드 상단에 `오늘/주간` 토글이 보인다.
- [ ] 최초 진입 시 기본 탭은 `오늘`이다.
- [ ] `오늘/주간` 전환 시 타이틀이 `오늘 TOP 10` / `주간 TOP 10`으로 맞게 바뀐다.
- [ ] `오늘/주간` 전환 시 랭킹 목록이 현재 모드 기준으로 다시 로드된다.
- [ ] 게임 종료 후 결과 배너의 `오늘 N위` 피드백은 기존처럼 daily 기준으로 동작한다.

## 3) API 기능
- [ ] `GET /api/rank?gameId=click10&mode=daily&periodKey=YYYY-MM-DD&sort=desc` 응답이 기존과 동일하다.
- [ ] `GET /api/rank?gameId=click10&mode=weekly&periodKey=2026-W14&sort=desc`가 정상 응답한다.
- [ ] `mode=weekly`에서 `periodKey`가 잘못되면 `400 invalid_query`를 반환한다.
- [ ] `sort`가 `asc/desc` 외 값이면 `400 invalid_query`를 반환한다.
- [ ] 주간 응답에서 동일 닉네임은 개인 최고 1개만 남는다.
- [ ] 동점일 때 `created_at` 빠른 기록이 먼저 정렬된다.

## 4) KST 주차 경계
- [ ] `2026-03-29 23:59 KST` 기록은 `2026-W13`으로 집계된다.
- [ ] `2026-03-30 00:01 KST` 기록은 `2026-W14`으로 집계된다.
- [ ] 경계 전후 기록이 같은 주간 랭킹에 섞이지 않는다.

## 5) 안정성/회귀
- [ ] 허브에서 10개 게임 랭킹 병렬 조회 시 체감 지연이 과도하지 않다.
- [ ] 네트워크 오류 시 기존 오류 메시지 경로(게임 랭킹)가 정상 동작한다.
- [ ] 기존 닉네임 저장/변경, PB 표시, 일간 랭킹 흐름에 회귀가 없다.

## 6) 권장 빠른 스모크 순서
1. 허브 접속 -> `오늘/주간` 토글 왕복 3회.
2. 임의 게임 1개 진입 -> 랭킹 토글 전환 확인.
3. 해당 게임 플레이 1회 -> 결과 배너 `오늘 N위` 확인.
4. 허브로 복귀 -> 같은 게임 카드에서 `오늘/주간` 순위 문구 확인.
